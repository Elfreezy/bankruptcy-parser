## Парсер сведений о банкротстве

**Цель**: Сбор данных о процедурах банкротства компаний по ИНН из официальных источников.

**Источники данных**

`https://fedresurs.ru/`

| ИНН | № дела | Дата |
|---|---|---|
| 23`*****`15  | А`***`   | 01.01.2025 |

`https://kad.arbitr.ru/`

| № дела | Наименование документа | Последняя дата |
|---|---|---|
| А`***`   | О зав`*****`а  | 01.01.2025 |

**Stack**: Python, PostgreSQL, Requests, BeautifulSoup, SQLAlchemy, Alembic, Docker

*Положение*: 
- После анализа ресурсов сбора информации средствами `DevTools` удалось найти основные api-сервисы при обращении к которым можно получить информацию в формате `JSON`;
- Для обработки информации формата `HTML` используется `BeautifulSoup`;
- `BaseUserAgent` включает в себя реализацию использования `proxy` в случае блокировок со стороны ресурса, но требует доработок т.к. бесплатные proxy крайне неэффективны;
- Схема БД имеет малую связность с целью записи максимального количества информации о попытках сбора информации в итерацию;
- Уникальносто ИНН проверяется на момент подачи в качестве параметра запуска;
- Этапы сбора информации предоставляются библиотекой `logging`.

---

### Быстрый старт

1. Необходимо создать файл переменных окружения (путь `app/.env`):
```
DATABASE_URL=postgresql://postgres:postgres@db:5432/postgres
```

2. Запустить сервисы (выполнение команд из корня проекта)
```
docker compose up -d
```

3. Разместить файл ИНН компаний по пути `src/<file_name>.xlsx`. Заменить `<file_name>` на имя файла

Структура файла:

| inn  |
|---|
| 23`*****`15  |
| ...  |
| 41`*****`23  |

4. Запуск приложения с параметрами. Заменить `<file_name>` на имя файла
```
docker compose run --rm parser python app/main.py src/<file_name>.xlsx
```
